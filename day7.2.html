Day 7.2：工程化重构 - Spring Boot 4.0 项目正式创建

架构进化 记录在 Acer Predator 上完成的仓库重命名、新仓库确立及项目初始化全过程。

1. 阵地迁移：旧仓库重命名

为了规范技术栈标识，正式将原 JavaScript 版学生管理系统更名，确立其作为“日志-1”的归档地位。

  • 旧名称：student-management-system
  • 新名称：student-management-system-JavaScript
# 修改本地 SSH 远程关联
git remote set-url origin git@github.com:sixiaopangai/student-management-system-JavaScript.git

2. 仓库确立：新项目命名与远程创建

确立 Java 学习阶段的核心生产力仓库，通过后缀明确区分开发栈。

  • 项目命名：student-management-system-Java
  • 创建策略：在 GitHub 网页端新建空仓库，暂不初始化 README。
  • 本地衔接：在 Acer Predator 的 02_Active 目录下执行 git clone 获取云端“集装箱”。
# 克隆新创建的空仓库
git clone git@github.com:sixiaopangai/student-management-system-Java.git

3. 核心过程：Spring Boot 4.0 项目注入

采用“原地生成”策略，利用 IntelliJ IDEA 的 Spring Initializr 为空仓库注入灵魂：

  • 环境规格：JDK 17 + Spring Boot 4.0.1 + Maven。
  • 依赖配置：Spring Web, JPA, MySQL Driver, Lombok, DevTools。
  • 逻辑：覆盖生成后，利用原有 .git 目录直接实现云端同步。

4. 填坑笔记：Git 环境深度实战

问题 A：git remote -v 没反应
修复：IDEA 初始化时重置了配置，需重新执行 git remote add origin...
问题 B：换行符警告 (LF vs CRLF)
解决：开启 core.autocrlf true 实现跨平台格式自动转换。
