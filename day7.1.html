Day 7.1：Java 全栈启航 - 双仓库建立与工程化初始化

架构进化 记录从 Node.js 动态栈向 Spring Boot 强类型栈的工程迁移过程。

1. “双轨并行”策略 (Dual-Track Strategy)

为了保持代码的纯净与日志的清晰，本次初始化确立了两个独立仓库：

  • 展示仓库 (secondweb)：存放 HTML 静态日志，托管于 second.yushenchuanmei.top。
  • 代码仓库 (sms-java)：存放 Spring Boot + Maven 源代码，作为生产力核心。

2. 本地初始化与 SSH 关联

在 Acer Predator 笔记本上，通过 JDK 17 环境启动项目，并利用已建立的 SSH 信任实现秒级同步。

# 初始化展示仓库
git init
git remote add origin git@github.com:sixiaopangai/secondweb.git
git push -u origin main
# 初始化 Java 系统代码仓库
git remote add origin git@github.com:sixiaopangai/student-management-system-java.git
git push -u origin main

3. 跨语言感悟：动态 vs 静态

维度 日志-1 (JavaScript/Node.js) 日志-2 (Java/Spring Boot)
类型安全 动态类型，运行才知错 强类型，编译期锁死 Bug
依赖管理 package.json (npm) pom.xml (Maven)
工程结构 扁平灵活 严谨的 Controller/Service/Mapper 层次

4. 部署里程碑

成功在 Cloudflare Pages 绑定 second.yushenchuanmei.top，标志着 Java 学习阶段的展示阵地正式建成。
